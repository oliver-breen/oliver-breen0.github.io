# Security Policy

## Security Audit Report

This document outlines the security measures implemented for Oliver Breen's cybersecurity portfolio website.

## Supported Versions

| Version | Supported |
|---------|-----------|
| Latest  | ✅        |

## Security Features Implemented

### 1. Dependency Security
- Regular dependency auditing using `bundler-audit`
- Minimal dependency footprint
- GitHub Dependabot enabled for automated security updates

### 2. Content Security
- No sensitive information exposed in public repository
- Professional contact information only
- External resources loaded securely via HTTPS

### 3. CI/CD Security
- GitHub Actions workflow with minimal required permissions
- Secure artifact handling
- Branch protection on main branch

### 4. Configuration Security
- Jekyll security configurations enabled
- SEO and sitemap plugins for professional presentation
- Secure theme usage from verified sources

## Reporting a Vulnerability

If you discover a security vulnerability in this portfolio site, please:

1. **DO NOT** create a public issue
2. Contact the owner through professional channels:
   - LinkedIn: [Oliver Breen](http://www.linkedin.com/in/oliver-breen-450a3058)
3. Provide detailed information about the vulnerability
4. Allow reasonable time for response and remediation

## Security Best Practices Followed

- ✅ HTTPS enforcement via GitHub Pages
- ✅ Minimal attack surface
- ✅ Regular dependency updates
- ✅ Secure CI/CD pipeline
- ✅ No sensitive data in repository
